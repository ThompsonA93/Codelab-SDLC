## Waterfall Model

The waterfall model is a linear development life cycle approach, where each phase must be completed in strict sequence. Typically you do not return to a previous phase without schedule penalties.

- Requirements: Gathering of all requirements and specifications in various forms.
- Design: Defines hardware and system specifications from the requirements.
- Implementation: Programming of the software in units, components and packages.
- Testing: Testing the entire, programmed system for faults and failures.
- Deployment: Deployment of the product into the customer environment.

The waterfall model is clear and predictable, but assumes that every aspect from problem definition to solution release is understood perfectly.

> Reference: [Managing the Development of Large Software Systems by Dr. Winston W. Royce](https://www.google.com/search?q=Managing+the+Development+of+Large+Software+Systems+by+Dr.+Winston+W.+Royce)


## Agile Methodology

Agile is the iterative approach to software development and project management. It focuses on delivering projects in small increments.
It describes the iterative sequence of all subsequent phases, performed in a fixed time frame:

- Plan
- Design
- Develop
- Test
- Deploy
- Review
- Repeat

> Reference: [Agile Software Development, Principles, Patterns, and Practices by Robert C. Martin.](https://www.google.com/search?q=Agile+Software+Development%2C+Principles%2C+Patterns%2C+and+Practices+by+Robert+C.+Martin)



### Scrum

Is one of several frameworks used to implement the Agile Workflow.
Here we describe sprints, regular, fixed-length time frames during which the project must be further developed. 

### Kanban

Kanban is the continuous implementation, framed in lead time and cycle time.
